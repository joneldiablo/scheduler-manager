Documentación API

Autenticación
MétodoEndpointDescripción
POST /api/auth/login Iniciar sesión. Body: {"username":"...","password":"..."}
POST /api/auth/logout Cerrar sesión
GET /api/me Obtener usuario autenticado
Tareas
MétodoEndpointDescripción
GET /api/tasks Listar todas las tareas
GET /api/tasks/{id} Obtener una tarea por ID
POST /api/tasks Crear una nueva tarea
PUT /api/tasks/{id} Actualizar una tarea existente
DEL /api/tasks/{id} Eliminar una tarea
Campos de una Tarea
CampoTipoDescripción
namestringIdentificador único de la tarea
labelstringNombre visible en el frontend
descriptionstringDescripción de la tarea
scriptstringURL HTTP o comando del sistema a ejecutar
schedule_datetimestring|nullFecha/hora ISO para ejecución programada
recursive_timestampint|nullIntervalo en milisegundos para repetición
expiration_datetimestring|nullFecha de expiración de la tarea
times_totalintNúmero máximo de ejecuciones (0 = ilimitado)
activeboolSi la tarea está activa (1) o inactiva (0)
Planificador (Buffer)
MétodoEndpointDescripción
GET /api/tasks/{id}/buffer Obtener las ejecuciones planificadas de una tarea
PUT /api/buffer/{id}/cancel Cancelar una ejecución pendiente
Trigger (Ejecución)
MétodoEndpointDescripción
POST /api/trigger/{id} Forzar ejecución inmediata de una tarea
POST /api/trigger/batch Ejecutar múltiples tareas pendientes
Historial
MétodoEndpointDescripción
GET /api/history Obtener historial de ejecuciones
Health
MétodoEndpointDescripción
GET /api/health Verificar estado del servidor
WebSocket

El servidor notifica eventos en tiempo real a través de WebSocket:

EventoDescripción
task_firedUna tarea fue ejecutada
buffer_updatedEl buffer de planificación se actualizó
task_updatedUna tarea fue creada, editada o eliminada
Formato de Respuesta

Todas las respuestas siguen la misma estructura:

{
  "success": true,
  "error": false,
  "status": 200,
  "code": 0,
  "description": "ok",
  "data": { ... }
}
Ejemplos de Uso
Crear una tarea (curl)
curl -X POST http://localhost:3000/api/tasks \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{
    "name": "mi-tarea",
    "label": "Mi Tarea",
    "script": "http://localhost:4000/health/ping",
    "schedule_datetime": null,
    "recursive_timestamp": 300000,
    "active": true
  }'
Listar tareas
curl http://localhost:3000/api/tasks \
  -H "Authorization: Bearer <token>"
